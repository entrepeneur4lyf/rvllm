23 Rust crates. 15 hand-written CUDA kernels. 16 MB binary. Zero errors across 14,620 verified requests. 8,339 tok/s. Matches vLLM at production batch sizes. dev start to finish in 22 hours by one guy and opus swarms. tokens & servers: $1,780.

opus was obviously the biggest supporter and trainwrecker of everything but man, what a model

The key insight: Python vLLM transfers the entire logit matrix (74 MB) from GPU to host every decode step for sampling. rvLLM runs argmax directly on GPU and transfers 512 bytes. That's a 150,000x reduction in PCIe traffic per step.

Head-to-head on same A100 80GB SXM4, Qwen2.5-1.5B, greedy, 32 tok/req:

N=1:   rvLLM 58    | vLLM 69
N=48:  rvLLM 3,436 | vLLM 2,887  -- rvLLM pulls ahead
N=64:  rvLLM 3,938 | vLLM 3,828  -- matched
N=128: rvLLM 6,301 | vLLM 6,400  -- matched
N=256: rvLLM 8,055 | vLLM 9,437
N=512: rvLLM 8,308 | vLLM 10,771
N=768: rvLLM 8,339 (peak)
N=1024: rvLLM 8,308 | vLLM 12,740

Startup: 6s vs 121s (20x faster)
Binary: 16 MB vs ~500 MB (31x smaller)
CPU RSS: 348 MB vs 1,033 MB (3x less)

Where the speed comes from:
- GPU-resident argmax (no logit transfer for greedy decoding)
- CUDA graph replay (entire forward pass = one driver call)
- FP16 hgemm with tensor cores + f16 KV cache
- Mixed prefill+decode batching (no pipeline bubbles)
- No GIL, no GC, no PyTorch tensor overhead
- Single cudaMalloc slab for KV cache -- zero per-request allocation
- Prefix caching for shared system prompts

Total cost: $1,500 in vast.ai GPU rentals + $280 Claude Code overage. Built with Claude Opus doing architecture, CUDA kernels, and debugging.
